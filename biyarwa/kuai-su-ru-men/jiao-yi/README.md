# 交易

当用户想要与 Biyachain 进行交互并进行状态变更时，他们会创建交易。一旦交易创建完成，它需要由与发起该状态变更的账户关联的私钥签名。签名完成后，交易会被广播到 Biyachain 网络。

在广播并通过所有验证（包括签名验证、值验证等）后，交易会被包含在一个区块中，并通过共识过程经过网络的批准。

### 消息

简单来说，消息是向 Biyachain 提供的关于期望状态变更的指令。消息是特定模块的对象，用于触发属于其模块范围内的状态转换。每个交易必须至少包含一条消息。

此外，多个消息可以打包在同一交易中。每个模块可用的消息可以在[相关章节](../../kai-fa-zhe/mo-kuai/)中找到。

### 交易上下文

除了消息，每个交易还有一个上下文。上下文包括费用(fees)、账户详情(accountDetails)、备注(memo)、签名(signatures)等信息。

### 交易流 <a href="#transaction-flow" id="transaction-flow"></a>

我们想要广播到 Biyachain 的每个交易都有相同的流程。这个流程包括三个步骤：准备、签名和广播交易。当交易被包含在区块中时，使用消息指定的状态变化将在 Biyachain 上应用。
